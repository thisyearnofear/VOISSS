## 🎤 Q&A Preparation

### Expected Questions & Answers

**Q: How do you handle payment verification across different chains?**

A: For EVM chains, we use x402 with EIP-712 signatures. For non-EVM chains, we've built the structure and will verify via RPC. OWS provides a unified wallet interface, so we can add chain-specific verification without changing the API.

**Q: What's the pricing model?**

A: Base pricing is $0.000001 per character. We apply chain-specific multipliers based on gas costs - Solana is 0.85x (cheapest), Ethereum is 1.1x (most expensive). Agents choose the most cost-effective chain.

**Q: How does this compare to traditional voice APIs?**

A: Traditional APIs require account creation, API key management, and subscriptions. With VOISSS + OWS, agents just need a wallet. They discover the API, make a request, pay, and get the voice - no human in the loop.

**Q: Is this actually being used?**

A: Yes! VOISSS is live at voisss.netlify.app with real users. We added OWS support for this hackathon, but the core voice generation and x402 payments were already working.

**Q: What chains do you support?**

A: We support 9 chains via OWS: Ethereum, Base, Arbitrum, Optimism, Polygon (EVM), plus Solana, Cosmos, TON, and XRP Ledger (structure ready). EVM chains are fully functional today.

**Q: How do you prevent abuse?**

A: Multiple security layers: agent verification (reverse CAPTCHA), rate limiting by tier, security scoring, and payment verification. Agents build reputation over time for higher rate limits.
